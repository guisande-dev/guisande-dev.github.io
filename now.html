Now

Construyo productos para aprender rápido y mantener el músculo técnico. Mi objetivo: pensar transversalmente, pero siempre terminar en “algo que se usa”.

Traino — “Designed by gymrats, for gymrats”

Nació por una necesidad real: ninguna app existente cubría mis requerimientos de tracking en gimnasio, y estaba cansado del Excel. La idea fue construir un producto desde cero sin background previo en desarrollo de producto, apoyándome en IA para acelerar aprendizaje y código.

  • Frontend: Streamlit (Python, multi-page)
  • Backend/DB: Supabase (PostgreSQL + Auth + Row Level Security)
  • Deploy: Streamlit Community Cloud

Próximo milestone: sumar features core y abrir pruebas con conocidos para feedback.

Product thinking Architecture Ship & iterate

SimplyFin — Personal finance dashboard

Dashboard para seguimiento financiero personal: ingresos y gastos, con visualización clara y escalable. Los datos se cargan en un Excel y la app los consume para construir un tablero interactivo.

  • Input: Excel (registro al momento del gasto)
  • Output: dashboard interactivo para insights, presupuestos y planificación

Aprendizaje clave: ordenar datos primero, y recién después buscar “insights”. (Sí, el orden también es una feature.)

ML Pipeline — Spotify API

Proyecto en progreso: pipeline de machine learning utilizando datos de canciones extraídos vía API de Spotify. Enfoque end-to-end: ingesta, feature engineering, entrenamiento y evaluación.
